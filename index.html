프로젝트 개요

혼자 사는 2030이 동네에서 가볍게 도움을 주고받을 곳이 있으면 좋겠다는 생각에서 시작한 지역 커뮤니티 앱입니다. 4인 팀으로 1개월 동안 만들었고, 저는 프론트엔드 구현과 Supabase 연동·배포를 맡았습니다.

기획 — 무엇을, 왜

동네 커뮤니티를 찾아보면 익명 게시판은 신뢰가 잘 안 가고, 오프라인 모임은 나가기가 부담스러웠습니다. 그 사이에서 동네 단위로 가볍게 신뢰를 쌓는 연결을 만들고 싶었어요. 그래서 도움을 주고받는 게시판, 동네 채팅, 내 방을 꾸미는 마이룸을 중심에 뒀습니다. 마이룸은 3D 모델 대신 PNG 가구를 배치하는 방식으로 개발 부담을 줄이면서 꾸미는 재미를 살렸어요.

주요 기능

온보딩(전화번호 인증 회원가입)부터 홈(동네 소식·모임), 게시판(글·모임 모집·랭킹·검색), 지도, 1:1·그룹 채팅, 친구·프로필, 마이룸, 출석·포인트·레벨, 마이페이지·설정, 알림·이벤트, 그리고 운영용 관리자 콘솔까지 만들었습니다. 화면이 아니라 기능 단위로 나눠서 작업했어요.

기술 스택

프론트는 React·Vite·TypeScript·Tailwind로 만들고, 백엔드는 Supabase 하나로 인증·DB·실시간·저장소를 처리했습니다. 배포는 Vercel을 써서 깃허브에 올리면 자동으로 배포되게 했고, 유입은 GA4와 UTM으로 봤어요. 관리자 콘솔은 따로 정적 페이지로, 사전등록 랜딩은 라이브러리 없이 단일 HTML로 만들었습니다.

  • React
  • Vite
  • TypeScript
  • Tailwind
  • Supabase
  • Vercel
  • GA4

아키텍처

앱·관리자 콘솔·랜딩 세 개가 Supabase 하나를 같이 씁니다. 테이블은 전부 RLS(행 단위 권한)를 켰고, 중요한 키는 서버에서만 쓰도록 했어요. 실시간 기능은 채팅·게시글·댓글에만 걸고, 저장소는 유저가 아무 이미지나 올리지 못하게 검수한 가구 PNG만 쓰게 했습니다.

가장 고생한 부분 — 로그인(Auth) 갈아끼우기

처음엔 anon 키로만 DB를 붙여서 서버가 요청한 사람이 누군지 몰랐습니다(auth.uid()가 없었어요). 그래서 ‘내 글만 수정·삭제’ 같은 권한을 걸 수가 없었습니다. 전화번호 로그인은 그대로 두고, 전화번호를 이메일 형태(01012345678@holo.app)로 바꿔 Supabase Auth에 등록한 뒤 users 테이블에 auth_id를 연결했어요. 이렇게 하니 닉네임으로 작성자를 찾던 기존 코드 1,137곳을 거의 안 건드리고 권한을 붙일 수 있었습니다. 실서비스 DB라 테스트 환경이 따로 없어서, 한 번에 바꾸지 않고 ‘고치고 → 올리고 → 확인하고 → 안 되면 되돌리기’를 반복하며 회원가입·로그인·권한을 하나씩 교체했습니다.

배포하면서

깃허브에 올리면 Vercel이 앱과 관리자 콘솔을 각각 자동으로 배포합니다. 사전등록 랜딩은 어디서 들어왔는지(UTM)를 저장해 뒀다가 나중에 채널별로 얼마나 왔는지 확인했어요. 사용자가 늘면 서버 비용이 얼마나 들지도 대충 계산해봤는데, 초반엔 거의 무료였고 만 명 정도부터 월 십몇만 원 수준이었습니다.

배운 점

RLS를 직접 만지면서 ‘누구인지(인증)’와 ‘뭘 할 수 있는지(권한)’를 코드가 아니라 DB에서 관리하는 걸 처음 해봤습니다. 그리고 실서비스에서는 기능을 새로 만드는 것보다, 잘못됐을 때 되돌릴 수 있게 조금씩 바꾸는 게 훨씬 중요하다는 걸 로그인 교체하면서 크게 느꼈어요.

Overview

내가 좋아하는 곡(MP3)을 올리면 박자를 분석해 노트를 자동으로 배치해주는, 브라우저에서 바로 즐기는 4레인 리듬 게임입니다. 설치·로그인 없이 링크로 실행되고 오프라인에서도 동작하는 PWA로, 기획·디자인(UX/UI)·프론트엔드까지 1인으로 완성했습니다.

Solution

프레임워크 없이 순수 HTML·CSS·JavaScript 단일 파일로 구현했습니다. Web Audio로 온셋·템포를 분석해 7단계 난이도별로 자동 채보하고, 판정·콤보·정확도·SS~D 랭크, 결과 타이밍 분포 그래프와 싱크 자동 보정까지 갖췄습니다.

개요

“내가 좋아하는 곡으로 바로 리듬 게임을 하고 싶다”는 생각에서 출발한 1인 프로젝트입니다. 업로드한 오디오를 분석해 박자에 맞는 노트를 자동 배치하고, 4개 레인 판정선에 노트가 닿는 순간을 맞춰 누르며 플레이합니다. 게임에 수록된 곡은 Suno AI로 직접 만들어 넣었습니다.

핵심 기능

자동 채보 · 업로드한 곡의 온셋·템포를 분석해 난이도에 맞춰 노트를 자동 배치

7단계 난이도 · BEGINNER → EASY → NORMAL → HARD → EXPERT → EXTREME → INSANE, 밀도·낙하 속도·롱노트 비중을 다르게 튜닝

판정 · 몰입 요소 · PERFECT/GREAT/GOOD/MISS 판정, 콤보·정확도·SS~D 랭크, 피버·라이프 게이지·미러/랜덤/히든/서든 모디파이어

결과 분석 · 기록 · 판정 타이밍 분포 그래프, 싱크 자동 보정, 곡별 베스트·레벨(XP)·업적

인터랙티브 마스코트 · PWA · 홈 캐릭터 반응 모션, 홈 화면 설치·오프라인 캐싱

디자인 & UX

기술 구현

Web Audio로 스펙트럴 플럭스 기반 온셋을 검출하고 박자 간격으로 템포를 추정해 비트 그리드에 스냅하는 방식으로 자동 채보합니다. requestAnimationFrame 게임 루프 위에서 Canvas로 파티클·이펙트를 렌더링하고 3D 원근 레인을 적용했으며, 고DPI 렌더링·오차 중앙값 기반 싱크 보정, Service Worker 오프라인 캐싱까지 외부 라이브러리 없이 단일 파일로 구현했습니다.

결과 & 배운 점

프레임워크 없이도 오디오 분석·게임 루프·PWA까지 하나의 완결된 웹 게임을 만들 수 있음을 확인했습니다. “직접 플레이하며 고치는” 루프가 기획서보다 빠르고 정확하게 UX 문제를 드러냈고, 캐릭터·색·모션·아이콘을 일관된 톤으로 묶었을 때 제품의 완성도와 인상이 크게 달라진다는 걸 배웠습니다.
